Redis 面试题 （Q only）

一、Redis基础概念

  1. Redis与Memcached的区别是什么？适用场景有哪些？
  2. Redis为什么读写性能高？单线程模型如何支撑高并发？
  3. Redis的过期键删除策略有哪些？
  4. 内存淘汰机制（LRU、LFU等）如何工作？

二、Redis数据结构与使用场景

  1. Redis支持哪些数据结构？各自的应用场景是什么？
  2. 如何用Redis实现分布式锁？需要注意哪些问题？
  3. Redis的HyperLogLog和布隆过滤器（Bloom Filter）的原理与使用场景？
  4. 如何用Redis实现延迟队列或消息队列？
  5. Redis的Sorted Set底层实现原理是什么？如何实现排行榜功能？

三、Redis持久化与高可用

  1. Redis 的持久化机制是什么？各自的优缺点？
  2. AOF重写（AOF Rewrite）的过程是怎样的？
  3. Redis主从复制的原理和数据同步流程？
  4. 哨兵（Sentinel）模式的故障转移流程是怎样的？
  5. Redis Cluster集群如何实现数据分片（Slot分配）和扩容？
  6. Redis Cluster的节点间通信机制（Gossip协议）是什么？

四、Redis事务与高级功能

  1. Redis事务（MULTI/EXEC）的ACID特性如何？与数据库事务有何区别？
  2. Redis管道（Pipeline）的作用是什么？如何提升性能？
  3. Lua脚本在Redis中的作用和优势？如何保证原子性？

五、缓存问题与解决方案

  1. 缓存穿透、缓存雪崩、缓存击穿的区别及解决方案？
  2. 如何保证缓存与数据库的数据一致性（如双写一致性）？
  3. 延迟双删策略的原理和实现步骤？
  4. 如何处理热Key和大Key问题？对性能的影响是什么？

六、性能优化与监控

  1. Redis的慢查询日志如何配置与分析？
  2. Redis内存碎片产生原因及优化方法？
  3. Redis的并发竞争问题（如多客户端写同一个Key）如何解决？
  4. Redis的网络模型（Reactor模式）和多路复用机制如何工作？

七、分布式场景与实战

  1. RedLock算法的实现原理及争议点是什么？
  2. Redis如何实现分布式会话（Session）管理？
  3. Redis与数据库结合时，如何设计缓存更新策略（Cache Aside/Read Through/Write Through）？

八、高级特性与源码

  1. Redis 6.0多线程模型是如何设计的？与单线程的区别？
  2. Redis的惰性删除（Lazy Free）机制是什么？
  3. Redis的持久化混合模式（RDB+AOF）如何配置？
  4. Redis的Stream数据类型如何实现消息持久化和消费组（Consumer Group）？

九、运维与安全

  1. Redis的常见性能监控指标有哪些（如QPS、内存使用率）？
  2. Redis如何实现数据备份与恢复？
  3. Redis的安全防护措施（如密码认证、禁用高危命令）？
