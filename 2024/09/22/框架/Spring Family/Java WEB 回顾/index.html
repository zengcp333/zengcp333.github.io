Java WEB 总结

一、基础组件与核心概念

Servlet

什么是Servlet

  • Servlet（Server Applet），全称Java Servlet。是用Java编写的服务器端程序，其主要功能在于交互式地浏览和修改数据，生成动态Web内容。狭义的Servlet是指Java语言实现的一个接口，广义的Servlet是指任何实现了这个Servlet接口的类，一般情况下，人们将Servlet理解为后者。
  • Servlet运行于支持Java的应用服务器中。从实现上讲，Servlet可以响应任何类型的请求，但绝大多数情况下，Servlet只用来扩展基于HTTP协议的Web服务器。

Servlet工作模式：

  • 客户端发送请求至服务器
  • 服务器启动并调用Servlet，Servlet根据客户端请求生成响应内容并将其传给服务器
  • 服务器将响应返回客户端

Servlet工作原理

  • Servlet接口定义了Servlet与servlet容器之间的契约。这个契约是：Servlet容器将Servlet类载入内存，并产生Servlet实例和调用它具体的方法。但是要注意的是，在一个应用程序中，每种Servlet类型只能有一个实例。
  • 用户请求致使Servlet容器调用Servlet的 Service() 方法，并传入一个ServletRequest对象和一个ServletResponse对象。ServletRequest对象和ServletResponse对象都是由Servlet容器（例如TomCat）封装好的，并不需要程序员去实现，程序员可以直接使用这两个对象。
  • ServletRequest中封装了当前的Http请求，因此，开发人员不必解析和操作原始的Http数据。ServletResponse表示当前用户的Http响应，程序员只需直接操作ServletResponse对象就能把响应轻松的发回给用户。
  • 对于每一个应用程序，Servlet容器还会创建一个ServletContext对象。这个对象中封装了上下文（应用程序）的环境详情。每个应用程序只有一个ServletContext。每个Servlet对象也都有一个封装Servlet配置的ServletConfig对象。

Servlet生命周期

当客户端首次发送第一次请求后，由容器(web服务器(tomcat))去解析请求， 根据请求找到对应的servlet，判断该类的对象是否存在，不存在则创建servlet实例，调取init()方法进行初始化操作，初始化完成后调取service()方法，由 service()判断客户端的请求方式，如果是get，则执行doGet()，如果是post则执行doPost()。处理方法完成后，作出相应结果给客户端。单次请求处理完毕。
当用户发送第二次以后的请求时，会判断对象是否存在，但是不再执行init()，而直接执行service方法，调取 doGet()/doPost()方法。 当服务器关闭时调取destroy()方法进行销毁。

四个过程:

  1. 实例化 –先创建servlet实例
  2. 初始化 –init()
  3. 处理请求 —service()
  4. 服务终止 –destory()

配置方式：

  • web.xml：通过 <servlet> 和 <servlet-mapping> 配置。
  • 注解：@WebServlet(urlPatterns = "/demo")（Servlet 3.0+）。

JSP (Java Server Pages)

  • 作用：动态生成 HTML，支持嵌入 Java 代码（<% ... %>、<%= ... %>）。
  • 内置对象：
    • request（HttpServletRequest）、response（HttpServletResponse）。
    • session（HttpSession）、application（ServletContext）。
    • pageContext、out（JspWriter）、config（ServletConfig）。
  • 作用域：
    • page → request → session → application（范围从小到大）。
  • JSTL & EL：
    • JSTL：标签库（如 <c:forEach>），简化逻辑代码。
    • EL：表达式语言（${user.name}），替代 <%= %>。

Filter（过滤器）

  • 作用：在请求到达 Servlet 前或响应返回客户端前进行拦截处理。
  • 常见场景：
    • 字符编码过滤（CharacterEncodingFilter）。
    • 权限验证、日志记录、跨域处理（CORS）。
  • 生命周期：init() → doFilter() → destroy()。
  • 配置方式：
    • web.xml：通过 <filter> 和 <filter-mapping>。
    • 注解：@WebFilter(urlPatterns = "/*")。

Listener（监听器）

  • 作用：监听 Web 应用中的事件（如上下文创建、Session 创建）。
  • 常见监听器：
    • ServletContextListener：监听应用启动/关闭。
    • HttpSessionListener：监听 Session 创建/销毁。
    • ServletRequestListener：监听请求创建/完成。
  • 配置方式：通过 <listener> 标签或 @WebListener。

二、请求与响应处理

1. HTTP 请求处理

  • 请求对象：HttpServletRequest
  • 获取参数：getParameter()、getParameterValues()。
  • 获取请求头：getHeader()。
  • 请求转发：request.getRequestDispatcher("/path").forward(request, response)。

2. HTTP 响应处理

  • 响应对象：HttpServletResponse
  • 设置状态码：setStatus()（如 404、500）。
  • 设置响应头：setHeader()、addHeader()。
  • 重定向：response.sendRedirect("/new-url")。

3. 文件上传与下载

  • 上传：

    • 使用 <form enctype="multipart/form-data">。
    • 通过 request.getPart("file")（Servlet 3.0+）或 Apache Commons FileUpload。

  • 下载：

三、会话管理与状态保持

1. Cookie

  • 作用：客户端存储少量数据（如 Session ID）。

  • 关键方法：

  • 属性：setMaxAge()（过期时间）、setPath()（作用路径）。

2. Session

  • 作用：服务端存储用户会话数据。

  • 关键方法：

  • Session 超时：通过 web.xml 配置 <session-config><session-timeout>30</session-timeout>。

3. Token 机制（JWT 等）

  • 替代 Session：无状态认证，适合分布式系统。
  • 流程：
    1. 用户登录后服务端生成 Token。、
    2. Token 存储到客户端（如 Cookie 或 LocalStorage）。
    3. 后续请求携带 Token，服务端验证有效性。

四、MVC 模式与架构设计

1. MVC 分层

  • Model：数据模型（如 JavaBean）。
  • View：视图（JSP、Thymeleaf、Freemarker）。
  • Controller：控制器（Servlet 或 SpringMVC 的 @Controller）。

2. 前后端分离

  • 核心思想：前端（HTML/JS）与后端（Java）独立开发，通过 API 交互。
  • 数据格式：JSON（通过 Jackson 或 Gson 序列化）。
  • 通信方式：RESTful API（使用 @RestController）。

3. RESTful 设计原则

  • 资源定位：URI 表示资源（如 /api/users/{id}）。
  • HTTP 方法：GET（查询）、POST（新增）、PUT（更新）、DELETE（删除）。
  • 状态码：200（成功）、201（创建）、400（客户端错误）、500（服务端错误）。

五、Web 安全与防护

1. 常见攻击与防御

  • XSS（跨站脚本攻击）：
    • 防御：对用户输入进行转义（如 HtmlUtils.htmlEscape()）。
  • CSRF（跨站请求伪造）：
    • 防御：使用 Token 验证（如 Spring Security 的 CsrfFilter）。
  • SQL 注入：
    • 防御：使用 PreparedStatement 或 ORM 框架（如 MyBatis、Hibernate）。

2. 安全配置

  • HTTPS：强制使用 HTTPS（配置 Tomcat 的 SSL 证书）。
  • 敏感信息：避免在 URL 或 Cookie 中传递敏感数据。

六、高级特性与优化

1. 异步处理（Servlet 3.0+）

  • AsyncContext：处理耗时操作（如长轮询）。

2. WebSocket

  • 作用：实现双向实时通信（如聊天室）。
  • 关键类：javax.websocket.Endpoint 或 Spring 的 @ServerEndpoint。

3. 性能优化

  • 静态资源缓存：设置 HTTP 缓存头（Cache-Control）。
  • 压缩响应：启用 GZIP 压缩（Tomcat 配置 compression="on"）。
  • CDN 加速：将静态资源托管到 CDN。

七、常见面试问题

Servlet 生命周期？与 Filter 的区别？

  • Servlet 生命周期：

    1. 初始化：init() 方法（仅一次，在 Servlet 实例化后调用）。
    2. 处理请求：service() 方法（每次请求调用，根据 HTTP 方法分发给 doGet/doPost）。
    3. 销毁：destroy() 方法（仅一次，在 Servlet 被卸载前调用）。

  • Filter 与 Servlet 的区别：

    • Filter：用于拦截请求/响应，在请求到达 Servlet 前或响应返回客户端前执行（如权限校验、日志记录）。
    • Servlet：直接处理请求并生成响应。

JSP 的九大内置对象有哪些？

对象名 类型 作用
request HttpServletRequest 封装客户端请求信息（参数、头、属性等）。
response HttpServletResponse 封装服务端响应信息（输出流、重定向等）。
session HttpSession 存储用户会话数据（跨请求）。
application ServletContext 存储全局应用数据（所有用户共享）。
pageContext PageContext 提供对页面作用域和其他内置对象的访问。
out JspWriter 向客户端输出内容（类似 PrintWriter）。
config ServletConfig 获取 Servlet 配置信息（如初始化参数）。
page Object 当前 JSP 页面的实例（相当于 this）。
exception Throwable 仅在错误页面（isErrorPage="true"）中可用，捕获异常信息。

Cookie 和 Session 的区别？如何解决分布式 Session 共享问题？

  • 区别：

    特性 Cookie Session
    存储位置 客户端（浏览器） 服务端（内存/数据库）
    安全性 较低（可能被篡改） 较高（数据在服务端）
    数据大小 有限制（约 4KB） 无限制（受服务端内存限制）
    生命周期 可设置过期时间 默认随浏览器关闭失效

  • 分布式 Session 共享方案：

    1. Session 复制：集群节点间同步 Session（性能差，不推荐）。
    2. 集中存储：将 Session 存储在 Redis/Memcached 等中间件中。
    3. Token 机制：使用无状态 Token（如 JWT），客户端存储 Token，服务端验证签名。

GET 和 POST 的区别？

  • 区别：

    特性 Cookie Session
    存储位置 客户端（浏览器） 服务端（内存/数据库）
    安全性 较低（可能被篡改） 较高（数据在服务端）
    数据大小 有限制（约 4KB） 无限制（受服务端内存限制）
    生命周期 可设置过期时间 默认随浏览器关闭失效

  • 分布式 Session 共享方案：

    1. Session 复制：集群节点间同步 Session（性能差，不推荐）。
    2. 集中存储：将 Session 存储在 Redis/Memcached 等中间件中。
    3. Token 机制：使用无状态 Token（如 JWT），客户端存储 Token，服务端验证签名。

Servlet 是线程安全的吗？为什么？

  • 线程不安全：Servlet 是单例多线程的，所有请求共享同一个实例。
  • 风险：如果 Servlet 中有实例变量（如 private int count;），多线程并发修改会导致数据不一致。
  • 解决方案：
    1. 避免使用实例变量，改用局部变量。
    2. 使用同步锁（synchronized），但会降低性能。
    3. 使用线程安全的数据结构（如 AtomicInteger）。

总结

  • 核心组件：Servlet、JSP、Filter、Listener。
  • 状态管理：Cookie、Session、Token。
  • 安全防护：XSS、CSRF、SQL 注入。
  • 架构设计：MVC、RESTful、前后端分离。
  • 高级特性：异步处理、WebSocket、性能优化。
