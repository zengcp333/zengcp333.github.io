HAProxy 配置详解与综合示例

一、配置文件结构解析

HAProxy配置文件由5个主要部分组成，优先级从高到低：

  1. global：全局系统参数
  2. defaults：默认参数模板
  3. frontend：客户端连接入口
  4. backend：服务端集群配置
  5. listen：frontend+backend组合声明

业务场景描述

某电商平台需要部署HAProxy实现以下需求：

  1. 同时支持HTTP和HTTPS访问
  2. 分离静态资源与动态请求
  3. 提供管理后台访问控制
  4. 实现灰度发布功能
  5. 监控与自动故障转移
  6. 防御CC攻击

配置解析与业务对应

  1. SSL/TLS配置：

    • 同时支持HTTP/2和HTTP/1.1
    • 使用2048位DH参数增强安全性

  2. 流量分类：

    • 静态资源路由到专用服务器（CDN架构）
    • API请求采用最少连接算法
    • 管理后台限制IP访问

  3. 灰度发布：

    • 通过路径/v2访问新版服务
    • Canary服务器权重控制流量比例

  4. 安全防护：

    • 连接速率限制防御CC攻击
    • URL长度和路径检测防注入

  5. 会话保持：

    • Cookie插入实现状态保持
    • 源IP哈希保障静态资源缓存命中

  6. 监控管理：

    • 独立统计端口带权限控制
    • 实时健康检查与自动切换
